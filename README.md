The problem I was solving throughout this course was designing and implementing a course planning application that could efficiently load course data, store it using an appropriate data structure, and allow users to search for courses and display them in alphanumeric order. The project required analyzing different data structures and selecting the one that best met the application's performance and functionality requirements.

2. I approached the problem by comparing vectors, hash tables, and binary search trees before deciding which structure would best satisfy the project requirements. Understanding data structures was important because each one has different strengths and weaknesses regarding searching, sorting, memory usage, and overall efficiency. By evaluating their runtime performance, I was able to make an informed decision and design a solution that balanced speed with usability.

3. One of the biggest roadblocks I encountered was debugging file-loading and build issues while developing the application. I overcame these problems by carefully testing each feature individually, verifying file paths and input validation, and using compiler error messages to identify and correct mistakes. Breaking the project into smaller pieces made it much easier to troubleshoot and confirm that each component worked before moving on to the next.

4. This project expanded my approach to software design by reinforcing the importance of planning before writing code. Creating pseudocode, analyzing runtime complexity, and considering multiple design options helped me think more critically about efficiency and organization. Instead of immediately writing code, I learned to evaluate different solutions and choose the one that best fits the application's requirements.

5. My work on this project also improved the way I write programs that are maintainable, readable, and adaptable. I organized the program into small, well-named functions, used meaningful variable names, added input validation and error handling, and separated different program responsibilities into logical sections. These practices make the code easier to understand, easier to modify, and easier to extend with new features in the future.
